# results/ — curated, tracked results

This directory holds **curated, human-meaningful results that are committed to
git**: metrics tables (CSV/Markdown), small figures, and a short README per
experiment. It is the organized, pushable record of what we found.

**Split of responsibilities (keep git lean):**
- `results/` (tracked) — curated tables, small figures (`*.png` here are allowed),
  and per-experiment summaries. Generated by scripts, never hand-transcribed.
- `outputs/` (gitignored, project storage) — raw run artifacts: checkpoints,
  full-res plots, logs, wandb. Indexed, not committed.
- `data/` (gitignored, project storage) — datasets + heavy intermediate artifacts
  (`data/MANIFEST.md` is the committed index).

## Layout
```
results/
  phase0/           # feasibility gates (extraction recall, timing, physics-signal ARI)
  extraction/       # HDG-adapter quality vs Tesseract reference
  taskT/  taskE/  taskL/    # per-arm result tables + figures
  ablations/        # §11.4 grid tables
  paper/            # final paper-ready tables/figures
```

## Rules
- One subdir per experiment/gate; include a short `README.md` stating config hash,
  git commit, seeds, and dataset manifest checksum (setup guide §6).
- Tables are emitted by scripts from tracked runs; `/paper-claims` checks them
  against the §12 hypothesis thresholds.
- Keep committed figures small; heavy/raw artifacts stay in `outputs/`.
